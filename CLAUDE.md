# The Architect

Infrastructure for declarative MCP agent workflows. Define workflows in Python, deploy on shared infra.

## Stack

- Python 3.12, uv as package manager
- FastAPI + FastMCP (MCP server)
- SQLAlchemy async + asyncpg + Alembic (database)
- Pydantic v2 + pydantic-settings (validation)
- Jinja2 (code generation templates)
- Click (CLI)
- structlog (observability)
- cryptography/Fernet (credential encryption)
- pgvector (agent experience - Phase 3)

## Commands

```bash
uv sync                    # Install dependencies
uv run pytest              # Run tests
uv run architect --help    # CLI
```

## Module Structure

```
src/architect/
  core/          # Config, database, observability, encryption
  primitives/    # FieldDef, EntityDefinition, WorkflowDefinition, etc.
  generators/    # Jinja2 templates -> models, schemas, tools, serialize, repo
  runtime/       # App factory, loader, pipeline engine, tool runner
  modules/       # Built-in: api_keys, approvals, state, executions, credentials, budgets
  providers/     # Integration providers (Instagram, Resend, etc.)
  cli/           # Click commands: init, plan, apply, serve, state, destroy
  generated/     # Output per workflow slug (auto-generated, do not edit)
```

## Conventions

- Module pattern: models.py, schemas.py, repository.py, service.py (per entity)
- MCP tools: register(mcp: FastMCP) with closures + get_session_factory()
- SQL tables: snake_case plural, prefixed by workflow slug for domain tables
- SQLAlchemy models: PascalCase
- Pydantic schemas: PascalCase + suffix (Create, Update, Read)
- Enums: StrEnum, SCREAMING_SNAKE values
- Tests: pytest + pytest-asyncio, asyncio_mode = "auto"
- Lint: ruff (line-length = 100), mypy strict
- Generated code has header: "# AUTO-GENERATED by The Architect -- do not edit"
